## Description

Build the Redemption Code modal — shown after a successful points redemption (triggered from the Redeem Points bottom sheet). Displays an animated checkmark, the redemption code in a styled code block (large monospace, accent border), the RM discount value, usage instructions, and a Done button that dismisses the modal and refreshes the points balance. Built as a variant of the `AppDialog` component from Phase 0.

## Scope

### In Scope
- Create or extend `AppDialog` with redemption code variant
- Animated checkmark (64px, accent color) — use `flutter_animate` fade + scale
- "Your Redemption Code" heading (heading-md, centered)
- Code block: large monospace font, accent border (1px), centered, format HEC-XXXX-XXXX
- "RM {X.XX} discount" (heading-sm, accent, centered)
- Body text: "Show this code to the counter staff. Valid for today's visit only." (body-sm, text-secondary, centered)
- "Done" primary button (full width) — dismisses modal, triggers points balance refresh callback
- Support dark mode

### Out of Scope
- Backend redemption API call and code generation (Process 11 Laravel)
- My Points screen balance refresh logic (Phase 9.1 handles callback)
- Redeem Points bottom sheet integration (Phase 9.2 provides trigger)

## Technical Spec

### Files to Create or Modify
- `lib/core/widgets/app_dialog.dart` — Add redemption code variant to existing AppDialog component
- Or create `lib/features/loyalty/redemption_code_modal.dart` as standalone widget if extending AppDialog is impractical

### API Endpoints
- Redemption code generated by Laravel backend (Process 11)

### Data / Schema
- Redemption code format: HEC-{4 alphanumeric}-{4-digit check} (e.g., HEC-A3F9-2024)
- Discount value: points_redeemed × 0.05 (RM)

### UI Components
- `AppDialog` — redemption variant (extend existing Phase 0 component)
- Animated checkmark — `flutter_animate` fadeIn + scaleIn on mount
- Code block — `Container` with accent border, large monospace `Text`
- `AppButton` (primary variant) — "Done" button

### Constraints
- All colors from `AppColors` tokens (no hardcoded hex)
- All typography from `AppTextStyles` (no hardcoded sizes)
- All spacing from `AppSpacing` constants
- Border radius from `AppRadius`, shadows from `AppShadows`
- Dark mode required
- Zero `FFButtonWidget` or `FlutterFlowTheme` references
- `flutter analyze` must pass with zero errors
